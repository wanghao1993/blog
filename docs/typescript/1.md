# 为什么要使用TypeScript

JS是弱类型的语言，经常在开发的过程中会看到很多类型导致的错误。例如

`Cannot read property, null is not object`

```js
例如：

function sayName (userInfo) {
    return userInfo.name
}

// 这时候是正常的
const res = requestApi
sayName({})

// 这时候可能没有拿到userInfo 可能就报错了
sayName('')
```
这是一个简单的例子，好像很好规避这种错误，但是当逻辑变得负责的时候就很难说了。

但是TS可以轻松帮助我们规避这种问题。

```typescript
function sayName (userInfo: { name: string }) {
    return userInfo.name
}

// 这时候是正常的

const res = await requestApi()
sayName(res)

// 这时候可能没有拿到userInfo 可能就报错了
// 如果没有很好的定义res的数据结构，此处就会有提示，
// sayName只能接受一个对象，且必须包含name属性
sayName(res)
```
但是Ts也是有缺点的，例如在大型项目中会增加编译负担，热更新会变得缓慢。

# 类型声明

## 基础类型

一共就这些，number, string, boolean, Array, tuple, undefined, null, enum, object, any, never, Symbol, BigInt, void等等，包含js的所有类型，且新增了一些

## 变量声明

```js
// 原始类型
let bool: boolean = true
let num: number | undefined | null = 123
let str: string = 'abc'
// str = 123

// 数组
let arr1: number[] = [1, 2, 3]
let arr2: Array<number | string> = [1, 2, 3, '4']

// 元组
let tuple: [number, string] = [0, '1']
// tuple.push(2)
// console.log(tuple)
// tuple[2]

// 函数
let add = (x: number, y: number) => x + y
let compute: (x: number, y: number) => number
compute = (a, b) => a + b

// 对象
let obj: { x: number, y: number } = { x: 1, y: 2 }
obj.x = 3

// symbol
let s1: symbol = Symbol()
let s2 = Symbol()
// console.log(s1 === s2)

// undefined, null
let un: undefined = undefined
let nu: null = null
num = undefined
num = null

// void
let noReturn = () => {}

// any
let x
x = 1
x = []
x = () => {}

// never
let error = () => {
    throw new Error('error')
}
let endless = () => {
    while(true) {}
}


// 数字枚举
enum Role {
    Reporter = 1,
    Developer,
    Maintainer,
    Owner,
    Guest
}
// console.log(Role.Reporter)
// console.log(Role)

// 1: "Reporter"
// 2: "Developer"
// 3: "Maintainer"
// 4: "Owner"
// 5: "Guest"
// Developer: 2
// Guest: 5
// Maintainer: 3
// Owner: 4
// Reporter: 1

// 字符串枚举
enum Message {
    Success = '恭喜你，成功了',
    Fail = '抱歉，失败了'
}

// {
// Fail: "抱歉，失败了"
// Success: "恭喜你，成功了"
// }

// 异构枚举
enum Answer {
    N,
    Y = 'Yes'
}
// 解析出来是这样的
// 0: "N"
// N: 0
// Y: "Yes"


// 枚举成员
// Role.Reporter = 0
enum Char {
    // const member
    a,
    b = Char.a,
    c = 1 + 3,
    // computed member
    d = Math.random(),
    e = '123'.length,
    f = 4
}

// 0: "b"
// 0.8325154981306795: "d"
// 3: "e"
// 4: "f"
// a: 0
// b: 0
// c: 4
// d: 0.8325154981306795
// e: 3
// f: 4

// 常量枚举
const enum Month {
    Jan,
    Feb,
    Mar,
    Apr = Month.Mar + 1,
    // May = () => 5
}
let month = [Month.Jan, Month.Feb, Month.Mar]

// 枚举类型
enum E { a, b }
enum F { a = 0, b = 1 }
enum G { a = 'apple', b = 'banana' }

let e: E = 3
let f: F = 3
// console.log(e === f) false 

let e1: E.a = 3
let e2: E.b = 3
let e3: E.a = 3
// console.log(e1 === e2) false
// console.log(e1 === e3) true

let g1: G = G.a
let g2: G.a = G.a

```
